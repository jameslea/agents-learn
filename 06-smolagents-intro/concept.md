# smolagents 第一印象：代码即控制流

在接触完 LangGraph 之后看 smolagents，最直观的感受是：**从“流程图”回到了“写程序”**。

## 1. 核心机制差异：Code vs. JSON

| 特性 | LangGraph | smolagents |
| :--- | :--- | :--- |
| **沟通方式** | 生成 JSON 结构化数据映射工具 | **生成 Python 代码片段** |
| **执行逻辑** | 由开发者在 Graph 中预设分支 (Nodes/Edges) | 由模型自主编写 Python 循环、变量和计算 |
| **灵活性** | 确定性极强，适合工业审批流 | 极其发散，适合解决数学、逻辑和动态问题 |
| **安全性** | 逻辑由你写，模型只做选择 (高) | 逻辑由模型写，需在沙箱中运行 (中) |

## 2. 为什么它叫 "smol"？
- **依赖少**：核心代码非常轻量，没有厚重的抽象层。
- **Docstring 驱动**：你不需要定义复杂的 Schema。只需写好标准的 Python 注释，模型就能读懂如何使用你的工具。

## 3. 适合什么场景？
不要用它来做严格的审批系统。但如果你需要一个 Agent 来：
- 分析 CSV 表格数据并画图。
- 解决复杂的数学证明题。
- 在多个工具之间进行极其复杂的动态逻辑转换。

`smolagents` 是目前的最佳选择之一。

# Agent 框架分类

**目标**：理解不同类型的 Agent 框架，以及它们在不同应用场景中的优势和适用场景。

**内容**：

Agent（智能体）框架是当前大模型应用开发的核心。根据**功能侧重点**、**架构设计理念**以及**应用场景**的不同，我们可以将现有的 Agent 框架分为四大类：**单体任务型、多智能体协同型（Multi-Agent）、自主探索型（Autonomous Agents）和底层基础设施/编排型**。

以下是详细的分类介绍：

---

### 1. 多智能体协同框架 (Multi-Agent Frameworks)
这类框架的核心思想是“分工”。通过模拟人类社会的组织架构，让不同的 Agent 扮演不同角色（如程序员、测试员、经理），通过对话和协作来解决复杂问题。

*   **AutoGen (Microsoft)**
    *   **特点：** 目前最流行的多智能体框架之一。它允许用户定义多个具有不同能力的 Agent，并设定它们之间的交互模式（如双人对话、群聊）。
    *   **核心优势：** 支持高度可定制的对话流；能够让 Agent 自动执行代码并观察结果；支持人类介入（Human-in-the-loop）。
    *   **适用场景：** 复杂的软件开发、自动化数据分析、需要多轮迭代的任务。
*   **MetaGPT**
    *   **特点：** 将“软件工程”的 SOP（标准作业程序）引入 Agent 框架。它不仅仅是对话，而是要求每个 Agent 遵循特定的文档规范（如 PRD、设计文档、架构图）。
    *   **核心优势：** 通过结构化的角色分工减少大模型的“幻觉”和逻辑混乱；强调输出物的标准化。
    *   **适用场景：** 软件工程自动化、需要严格流程控制的任务。
*   **ChatDev**
    *   **特点：** 模拟了一个虚拟的软件公司，Agent 分为 CEO, CTO, Programmer, Reviewer 等角色。
    *   **核心优势：** 实验性强，展示了 Agent 协作完成小型项目（如写一个贪吃蛇游戏）的全过程。

---

### 2. 自主探索型/通用任务框架 (Autonomous Agents)
这类框架的核心是“自主性”。给定一个长远目标，Agent 会通过“感知 $\rightarrow$ 规划 $\rightarrow$ 行动 $\rightarrow$ 反思”的循环，自行决定下一步该做什么。

*   **BabyAGI / AutoGPT**
    *   **特点：** 这一类框架的鼻祖。它们的核心逻辑是任务列表管理（Task Management）。Agent 会根据目标生成任务清单，执行一个，然后根据结果更新清单。
    *   **核心优势：** 完全不需要人工干预，具备极强的“自我驱动”感。
    *   **适用场景：** 市场调研、自动化的 Web 搜索、简单的自动化流程探索。
*   **Voyager (基于 Minecraft 的研究型框架)**
    *   **特点：** 展示了 Agent 如何通过“技能库（Skill Library）”进行终身学习。它通过观察环境并编写 JavaScript 代码来保存新学到的能力。
    *   **核心优势：** 提出了“代码即技能”的概念，展示了 Agent 的进化能力。

---

### 3. 底层编排与工具集成框架 (Orchestration & Tool-use)
这类框架更像是“开发者工具箱”。它们不强调复杂的社会化协作，而是侧重于如何让大模型稳定地调用 API、连接数据库和管理上下文。

*   **LangChain**
    *   **特点：** Agent 开发的行业标准。它提供了极其丰富的组件（Chains, Memory, Retrievers, Tools）。
    *   **核心优势：** 生态系统庞大；可以将 LLM 与任何外部数据源（PDF、SQL、Google 搜索）无缝连接。其 `LangGraph` 模块目前正成为构建复杂循环 Agent 的主流手段。
    *   **适用场景：** 构建 RAG（检索增强生成）应用、定制化工作流、企业级知识库。
*   **LlamaIndex**
    *   **特点：** 专注于“数据”与“Agent”的结合。它不仅是 Agent 框架，更是数据索引专家。
    *   **核心优势：** 在处理大规模私有数据（Unstructured Data）并将其转化为 Agent 可理解的知识方面表现最强。
    *   **适用场景：** 基于企业内部文档的智能问答、高精度知识检索型 Agent。

---

### 4. 低代码/可视化 Agent 平台 (No-Code / Low-Code Platforms)
这类框架面向非算法工程师或希望快速原型设计的开发者，通过拖拽 UI 来构建逻辑。

*   **Coze (字节跳动 - Coze.cn / Coze.com)**
    *   **特点：** 集成了插件、工作流（Workflow）、知识库和长短期记忆。
    *   **核心优势：** 极低的门槛；拥有极其丰富的生态插件（搜索、图像生成等）；一键发布到微信、Discord 等平台。
*   **Dify.ai (开源)**
    *   **特点：** 开源的 LLM 应用开发平台，集成了 Prompt 工程、RAG 和 Agent 编排。
    *   **核心优势：** 对后端逻辑（Workflow）的可视化支持非常好；适合企业内部构建私有的 Agent 应用。

---

### 总结对比表

| 框架类别 | 代表框架 | 核心关键词 | 适合人群 | 解决的核心问题 |
| :--- | :--- | :--- | :--- | :--- |
| **多智能体协同** | AutoGen, MetaGPT | 分工、SOP、对话流 | 算法工程师 | 如何让多个角色协作完成复杂项目 |
| **自主探索型** | AutoGPT, BabyAGI | 目标驱动、自我进化 | 研究者/极客 | 如何实现无人值守的自动化任务 |
| **编排与工具集成** | LangChain, LlamaIndex | RAG、工具调用、数据连接 | 后端/AI 工程师 | 如何让 LLM 连接外部世界和私有数据 |
| **低代码平台** | Coze, Dify | 拖拽、工作流、快速部署 | 产品经理/开发者 | 如何快速落地一个可用的 AI 应用 |

**建议选择路径：**
*   如果你要开发**企业知识库问答** $\rightarrow$ 选择 **LlamaIndex** 或 **Dify**。
*   如果你要模拟**软件开发团队** $\rightarrow$ 选择 **AutoGen** 或 **MetaGPT**。
*   如果你想做一个**自动化搜索/调研工具** $\rightarrow$ 参考 **AutoGPT** 的逻辑或使用 **LangChain Agent**。
*   如果你只是想**快速验证想法** $\rightarrow$ 使用 **Coze**。

# 第二章实战任务书：高级检索增强 (Advanced RAG)

## 🎯 核心目标
通过代码实测，验证 **Rerank (重排序)** 和 **HyDE (查询扩展)** 在解决“语义混淆”和“模糊提问”时的真实威力。

## 🛠️ 下一步开发任务

### 1. 构造“语义混淆”数据集 (`practice/dataset.py`)
- 设计 10-20 条容易误导向量检索的文档片段（如：医疗术语相似但病理不同的段落）。
- 定义标准测试集：包含“模糊提问”和“标准答案”。

### 2. 编写对比评测脚本 (`practice/benchmark_rag.py`)
- **实现模式 1：Baseline**
    - 使用 `bge-small-zh-v1.5` 进行向量化。
    - 使用 FAISS 进行 Top-K 检索。
- **实现模式 2：Rerank 增强**
    - 接入 `bge-reranker-v2-m3` 模型。
    - 对 Baseline 返回的 Top 20 结果进行二次精排。
- **实现模式 3：HyDE 联想**
    - 调用 Ollama 或 LM Studio (**Qwen 3.6 35B** 或 **Gemma 4 26B**) 生成假设性回答。
    - *注：在具备足够显存的高性能环境下，优先使用 30B+ 级别模型以获得更精准的语义扩展。*
    - 用该回答的向量去检索库。

### 3. 性能与精度指标观察
- 记录三种模式在实际运行设备上的推理耗时与显存/内存占用。
- 观察正确文档在结果列表中的排名（Hit Rate @ 1 / @ 3）。

## 📝 预习知识点
- 了解 `Cross-Encoder` 为什么比 `Bi-Encoder` 慢但准。
- 思考 HyDE 在什么情况下会产生“反作用”（比如：模型编得太离谱）。
- **2026 深度洞察**：对比轻量级模型（7B）与中大型模型（30B+）在 HyDE 任务中的“幻觉抑制”表现及其对检索准确率的影响。

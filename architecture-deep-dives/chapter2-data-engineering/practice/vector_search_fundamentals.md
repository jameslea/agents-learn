# 向量搜索与索引 (Vector Search & Indexing) 基础指南

这是一份专为开发者准备的向量技术入门指南，涵盖了从“语义理解”到“工业级检索”的核心知识体系。

---

## 1. 核心概念：什么是向量化 (Embedding)？

在 AI 的世界里，计算机不直接阅读文字，而是将文字转化为一串数字，即**向量 (Vector)**。
- **过程**：文本 -> Embedding 模型 -> 一串数字（如 `[0.12, -0.05, 0.88, ...]`）。
- **本质**：将模糊的“语义”映射到高维坐标系中的一个“点”。
- **神奇之处**：含义相近的词或句子，在坐标系里的距离也会更近。例如，“国王”与“王后”的距离，比“国王”与“香蕉”的距离要近得多。

## 2. 相似度计算：如何衡量“近”？

有了坐标点，我们需要数学公式来计算它们之间的距离。主流有三种方式：

| 计算方式 | 适用场景 | 简单理解 |
| :--- | :--- | :--- |
| **余弦相似度 (Cosine)** | **最常用** | 计算两个向量之间的**夹角**。角度越小，语义越一致。不受文本长度影响。 |
| **欧氏距离 (L2)** | 科学计算、聚类 | 计算两点之间的**直线距离**。距离越小越相似。代码中 FAISS 默认常用。 |
| **内积 (Inner Product)** | 推荐系统、大规模检索 | 向量对应位置相乘再相加。通常用于已经归一化的向量。 |

## 3. 索引算法：如何从海量数据中“瞬间”找到结果？

当你有 1000 万条文档时，逐一对比（暴力搜索）太慢了。我们需要**索引 (Indexing)** 技术，也就是 **ANN (近似最近邻)** 算法。

### 3.1 Flat (暴力搜索)
- **原理**：不建立任何索引，挨个算距离。
- **优缺点**：100% 精确，但速度最慢。适用于 1 万条以下的小规模数据。
- **代码体现**：`faiss.IndexFlatL2`。

### 3.2 IVF (倒排索引)
- **原理**：先用聚类算法把向量分成很多“桶”。搜索时，先判断问题属于哪个桶，只去那个桶里找。
- **比喻**：像去图书馆找书，先根据“分类号”找到书架，再在书架上找。

### 3.3 HNSW (分层导航小世界图) —— **目前的工业标准**
- **原理**：把向量连成一张“社交网”。通过层层跳转，快速逼近目标。
- **优缺点**：极快、极准，但占内存较多。是目前 Pinecone, Milvus 等数据库的核心算法。

### 3.4 PQ (乘积量化)
- **原理**：对向量进行“高倍压缩”。
- **比喻**：把高清图片存成低像素缩略图，牺牲一点精度来换取巨大的内存节省。

## 4. 数据工程：检索效果的“幕后英雄”

检索得准不准，不仅仅看算法，更看数据怎么喂：

1.  **切分 (Chunking)**：
    - 文档太长（如 50 页 PDF）无法整体向量化，必须切块。
    - **金律**：切块太小丢失上下文，切块太大语义被稀释。
2.  **重叠 (Overlap)**：
    - 切块时，块与块之间保留 10%~20% 的重复内容，防止重要信息恰好被“切断”。
3.  **混合检索 (Hybrid Search)**：
    - **向量搜索**擅长“模糊语义”，**关键词搜索**擅长“精准名称/编号”。
    - **最佳实践**：两者结果加权合并。

## 5. 工具链选型建议

- **本地开发/小规模**：`FAISS` (Facebook 开源，轻量级，性能巅峰)。
- **Agent/简单 Demo**：`ChromaDB` (易用性极高，Python 一键安装)。
- **生产级/亿级数据**：`Milvus`, `Qdrant`, `Pinecone` (分布式，高可用)。

---
**学习心法**：向量搜索是 RAG 的“眼睛”。先学会用 FAISS 建立索引（看清数据），再学会用 Rerank 做二次校准（看准数据），你就能构建出工业级的 AI 应用。

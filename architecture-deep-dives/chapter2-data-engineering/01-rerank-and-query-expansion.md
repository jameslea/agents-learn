# 深度探索 2.1：从基础 RAG 到高级检索链 (Advanced Retrieval)

> **探索目标**：解决基础 RAG 系统中“查不准”、“漏关键信息”的痛点。Agent 的智力上限取决于它摄入的数据，我们将探讨如何通过 Rerank 和 Query Expansion 榨干检索库的价值。

---

## 一、 为什么传统的向量检索（基础 RAG）会失败？

在早期的 Agent 开发中（如你在阶段 10 之前做的），我们通常把文档切块（Chunking），然后用 Embedding 模型转成向量，存入 Chroma 或 Milvus，最后通过“余弦相似度”来寻找答案。

**这种基于 Bi-Encoder（双编码器）架构的缺陷：**
1. **字面相似度陷阱**：当你搜“苹果手机怎么退货”时，向量检索可能会给你推一篇“苹果怎么种植能高产”的文章，因为它们在词汇向量空间中有重叠，但语义完全无关。
2. **颗粒度太粗**：把 500 个字压缩成一个 768 维的向量，必然会丢失大量细节（这就是所谓的“信息塌缩”）。

---

## 二、 破局方案一：Rerank (重排序) 机制

**Rerank 是目前提升 RAG 准确率性价比最高的技术，没有之一。**

### 1. 工作原理：Cross-Encoder (交叉编码器)
传统的向量检索是“瞎子摸象”，而 Rerank 使用的是 Cross-Encoder 架构。
- **初筛（粗排）**：先用普通的向量检索，快速从 100 万篇文档中捞出最可能相关的 Top 50（这一步为了**速度**）。
- **精排（Rerank）**：将用户的提问（Query）与这 50 篇文档逐一“拼”在一起，让 Rerank 模型（如 `bge-reranker` 或 `Cohere Rerank` API）进行极其细腻的字对字注意力计算，打出 0-1 的精确相关性得分。
- **截断**：最后只取最高分的 Top 5 喂给 Agent。

**👉 结果**：彻底解决了字面相似但语义无关的“幻觉”文档干扰 Agent 思考的问题。

---

## 三、 破局方案二：Agentic 检索之 Query Expansion (查询扩展)

很多时候搜不到答案，不是知识库里没有，而是用户（或主控 Agent）的提问太模糊。

### 1. 核心痛点
用户提问：“特斯拉最近有什么负面新闻吗？”
如果你直接拿这句话去向量库里搜，可能什么都搜不到，因为新闻稿里写的是“刹车失灵”、“自动驾驶事故”或“产能不达标”，根本没有“负面新闻”这个词。

### 2. 查询扩展的三种高级玩法
我们需要让一个专门的 Agent（或 LLM 节点）在真正检索前，对提问进行“手术”：

*   **多角度改写 (Multi-Query)**：
    Agent 将原问题重写为三个不同的搜索词：
    1. "特斯拉 召回 事件"
    2. "特斯拉 自动驾驶 事故 2023"
    3. "Tesla stock drop reasons"
    然后并发搜索，合并结果后再去重。
*   **后退提问 (Step-back Prompting)**：
    遇到具体问题搜不到时，让 Agent 往宏观退一步。比如问“物理学家爱因斯坦在1905年上了哪所大学”，先让 Agent 改写为“爱因斯坦 1905年 个人履历”，先获取宽泛背景，再从中提取细节。
*   **HyDE (假设性文档嵌入)**：
    **这是最惊艳的技巧**。对于用户的提问，先让大模型“瞎编”一个答案，然后把这个“瞎编的答案”转换成向量，去知识库里找最相似的真实文档。因为答案和答案在向量空间的结构更接近，这往往能绕过“提问和文档结构不匹配”的鸿沟。

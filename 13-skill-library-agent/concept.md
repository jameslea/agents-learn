# 技能库与长期学习：Voyager 的核心思想

> 本文档探讨如何让 Agent 拥有“长期记忆”和“技能沉淀”的能力，实现从“新手”到“专家”的自我演进。

---

## 一、 什么是“技能” (Skill)？

在 Voyager 框架中，一个“技能”不再是一段模糊的对话描述，而是一个**可复用的结构化实体**。它通常包含：

1.  **标识符 (ID/Name)**：技能的唯一名称（如 `web_search`）。
2.  **语义描述 (Description)**：这个技能是干什么的，在什么场景下使用。
3.  **参数定义 (Schema)**：输入什么，输出什么。
4.  **实现逻辑 (Implementation)**：一段可执行的代码、一个工具函数，或者一段经过验证的复杂 Prompt。

## 二、 技能模式的工业溯源

除了学术界的 Voyager，工业界对“技能库”的探索同样深远：

1.  **起源与风靡**：这种模式最早由 **Anthropic (Claude)** 大力推广。通过为模型增加显式的技能支持，让 Agent 具备了跨越式能力。随后，这一模式风靡全球，成为 Dify、Coze 等主流 Agent 平台的标配。
2.  **技能的本质：结构化提示词**：在实战中，一个“技能”往往不只是一段代码，它更像是一段**经过精心调试的结构化提示词**。它通常拥有固定的模板，包含：
    *   **流程指令**：规定 Agent 解决该问题的步骤。
    *   **强约束条件**：限制 Agent 的行为边界，减少幻觉。
    *   **可用工具挂载**：明确该技能可以调用的外部 API。
3.  **核心价值**：这种“提示词即技能”的模式，将不可控的 LLM 输出变成了**可复用、可工程化管理的资产**。

---

## 三、 技能库的工作流 (The Skill Loop)

一个具有长期学习能力的 Agent 遵循以下循环：

### 1. 任务前：技能检索 (Retrieval)
当接收到任务时，Agent 首先去自己的“技能库”中搜索。
*   **向量搜索**：利用问题的语义，在技能库中寻找最相关的“技能描述”。
*   **上下文注入**：将找到的技能代码/文档注入到当前的 Prompt 中，告诉 LLM：“你已经学过这个了，直接用这段逻辑。”

### 2. 任务中：技能应用 (Execution)
Agent 尝试运行技能。如果成功，任务完成；如果失败，进入“调试模式”，利用 LLM 修复逻辑。

### 3. 任务后：技能沉淀 (Synthesis)
这是最关键的一步。如果 Agent 发现自己通过一套新的逻辑解决了一个以前没遇到过的问题，它会：
*   **总结**：将这套逻辑提炼成一个通用的“新技能”。
*   **存档**：将其存入数据库，供下次使用。

---

## 三、 为什么这比单次的 Agent 更强大？

*   **克服遗忘**：LLM 的上下文是有限的，但数据库是无限的。
*   **效率递增**：随着技能库的充实，Agent 解决复杂问题的速度会越来越快，因为它是在“读档”而不是“重写”。
*   **可解释性**：你可以清晰地看到 Agent 已经掌握了哪些技能，甚至可以手动修正库里的某个技能。

## 四、 本阶段实验目标

我们将构建一个简易的 `SkillLibraryAgent`：
1.  实现一个基于本地 JSON 的技能库。
2.  演示 Agent 如何根据任务自动检索技能。
3.  演示 Agent 在完成新任务后，如何将经验沉淀为“新技能文件”。

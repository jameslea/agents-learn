## 二、 核心观察：技能库 vs. 记忆

| 维度 | 普通记忆 (Memory) | 技能库 (Skill Library) |
| :--- | :--- | :--- |
| **存储内容** | 聊天历史、零散事实 | **可执行的逻辑/代码** |
| **复用方式** | 语义检索历史片段 | **精准匹配功能函数** |
| **进化感** | 知道“聊过什么” | 知道“能做什么” |

在 `skill_library_agent.py` 中，我们看到了 Agent 如何在完成任务后，通过 `NEW_SKILL_FOUND` 标记自主触发“技能提取”逻辑。

## 三、 待验证问题

1.  **技能膨胀**：如果 Agent 产出了成千上万个细碎技能，检索效率会如何？
2.  **技能更新**：如果旧技能有 Bug 或不再适用，Agent 如何自主更新库中的代码？
3.  **抽象层次**：Agent 总结的技能是太宽泛（无法复用）还是太具体（只能用一次）？

---

- **技能库 vs. 简单 RAG**：RAG 检索的是“资料”，Agent 还需要自己想怎么用；技能库检索的是“方案/代码”，Agent 可以直接“拿来主义”，效率和准确度更高。
- **自我进化能力**：通过本次实验（斐波那契数列任务），Agent 成功将复杂的迭代逻辑沉淀为 `迭代动态规划计算斐波那契数列` 技能，并进一步推导出 `前 n 项和` 公式。这证明了 Agent 具备通过任务反馈不断充实自身工具箱的能力。
- **长期价值**：这种机制是实现“通用人工智能 (AGI)”的关键拼图之一——让 AI 不再是每次重启都失忆的对话框，而是真正能够通过实践不断成长的“老师傅”。
